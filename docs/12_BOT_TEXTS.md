# 12 BOT_TEXTS, все тексты бота для пользователей

Единый источник правды для всех строк, которые бот шлёт пользователю. Все строки на **русском**.
Используются как константы из `supabase/functions/_shared/texts.ts`. Менять тексты только тут +
перегенерация `texts.ts`.

Правила:

- Без em-dash (`-`). Используй обычный hyphen `-` или запятые.
- Эмодзи допустимы для эмоциональной разметки, не злоупотреблять.
- Markdown V2 экранирование при необходимости (символы `_*[]()~>#+-=|{}.!`).
- Плейсхолдеры: `{name}`, `{amount}`, `{currency}`, и т.д.

## 1. Команды

### /start (член семьи)

```
Привет, {name}. Я FinBot, помогаю семье вести учёт трат.

Просто шли мне:
- текстом: "купил кофе 12 zł"
- голосовым сообщением
- фото чека (jpeg или heic)

Полная справка: /help.
```

## 2. Подтверждение записи

### Обычная запись текстом или голосом

```
✅ Записал.

{name}: {amount} {currency} ({category})
{date}
```

С inline кнопками:

- `↩️ Отменить` (callback `undo:<expense_id>`)
- `✏️ Категория` (callback `cat_menu:<expense_id>`)

### Голос: прогресс-этапы

1. `🎙 Распознаю...` (editMessageText сразу после download)
2. `🤖 Думаю...` (после Whisper)
3. Финальное подтверждение (после Claude + DB insert), как выше.

### Несколько записей в одном сообщении

```
✅ Записал {count} трат:

1. {name1}: {amount1} {currency1} ({cat1})
2. {name2}: {amount2} {currency2} ({cat2})
...

Итого: {total} zł
```

### Чек по фото (сводка)

```
🧾 *Чек {merchant}, {date}*

Итого: {total} {currency}
Распознано {count} позиций.

По категориям:
- {cat1}: {sum1} zł ({n1})
- {cat2}: {sum2} zł ({n2})
- ...
```

Inline кнопки:

- `📋 Подробно` (callback `receipt_details:<receipt_id>`)
- `↩️ Отменить` (callback `undo_receipt:<receipt_id>`)

### Чек: подробно

```
*Чек {merchant}, {date}*

1. {item1_name} - {item1_amount} {currency} ({cat1})
2. {item2_name} - {item2_amount} {currency} ({cat2})
...
```

### Чек с расхождением (reconciliation failed)

```
🧾 *Чек {merchant}, {date}*

⚠️ Расхождение позиций и итога больше 5%, пометил для ручной проверки. Можешь поправить через "✏️ Категория" или "↩️ Отменить".

Итого по чеку: {total} {currency}
Сумма позиций: {items_sum} {currency}
Распознано: {count} позиций.
```

## 3. Подтверждение больших сумм

```
⚠️ Большая сумма: {name} {amount} {currency} ({amount_pln} zł).

Подтверждаешь?
```

Inline кнопки:

- `✅ Да` (callback `conf_yes:<expense_id>`)
- `✏️ Изменить` (callback `conf_edit:<expense_id>`)
- `❌ Отмена` (callback `conf_no:<expense_id>`)

Через 60 сек если ничего не нажато: бот editMessageText на:

```
✅ Записано автоматически после 60 сек: {name} {amount} {currency}.
```

После `conf_yes`:

```
✅ Подтверждено: {name} {amount} {currency}.
```

После `conf_no`:

```
❌ Запись отменена.
```

## 7. Ошибки

### Telegram lang detection не whitelist (Whisper)

```
Не понял язык голосового сообщения: {detected_lang}. Поддерживаются: русский, украинский, польский, английский. Попробуй ещё раз или напиши текстом.
```

### Voice слишком длинное (> 5 мин)

```
Голосовое больше 5 минут, не обрабатываю. Сделай покороче или несколько голосовых.
```

### Voice не удалось распознать (Whisper API error)

```
Не получилось распознать голосовое. Попробуй ещё раз или напиши текстом.
```

### Фото слишком тёмное или blurry (Claude vision вернул error)

```
Чек плохо читается. Попробуй сфотографировать снова при лучшем свете, выровняй камеру параллельно бумаге.
```

### Фото слишком большое (> 20 MB после конверсии)

```
Фото слишком большое. Пришли в меньшем размере или сожми.
```

### Не понял что это за трата (Claude вернул без tool_use)

```
Не понял, это про трату? Если да, перефразируй: "купил {что-то} за {сумма}".
```

### Дневной бюджет per-user превышен (soft)

```
⚠️ Ты сегодня потратил больше {budget}$ в ИИ-операциях бота. Продолжаю работать, но будь экономнее. Лимит сбросится в полночь.
```

### Дневной бюджет global превышен (hard)

```
🚫 Дневной бюджет на ИИ исчерпан. Бот возобновит работу в 00:00. Извини.
```

### Внешний сервис недоступен (Anthropic 5xx, retry будет позже)

```
🐢 Загружено, секунду... (повторю через минуту)
```

### Pending retry успешно прошёл

```
✅ Записал отложенную трату: {name} {amount} {currency}.
```

### Pending retry окончательно провалился (после 5 попыток)

```
❌ Не получилось обработать трату от {date}: {reason}. Перешли заново или напиши текстом.
```

### Storage upload failed (фото)

```
❌ Не удалось сохранить фото. Попробуй ещё раз.
```

### Internal error (catch-all)

```
❌ Что-то пошло не так. Уже работаю над этим.
```

Параллельно админу: `🚨 Internal error в {function}: {error_message}, request_id={req_id}`.

### /health backup-confirm

```
Подтверждаешь что age private key (BACKUP_ENCRYPTION_KEY) находится в безопасном месте (1Password, etc.)? Без него восстановление невозможно.
```

Inline кнопки:

- `✅ Да, ключ в безопасности`
- `❌ Нет, отложу`

После `Да`:

```
✅ Подтверждено. Со следующей субботы бэкапы начнут записываться.
```

После `Нет`:

```
Окей, повтори когда будешь готов. Бэкапы пока не пишутся, в логах будет предупреждение каждую субботу.
```

### /audit <id>

```
2. {timestamp}: update_category by {member} - {cat1} -> {cat2}
```
