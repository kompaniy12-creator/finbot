# Resume prompt, для возобновления сессии Claude Code

Используется когда:

- Сессия Claude Code умерла (контекст исчерпан, сетевой обрыв, авария).
- Локальная машина перезагрузилась.
- Серхий специально прервал работу и хочет продолжить позже.

## Условия для resume

1. `docs/STATE.md` существует и заполнен.
2. Рабочая директория на месте, git история не повреждена.
3. `.env` существует со всеми ключами.
4. Доступ к Supabase / GitHub / Telegram сохранён (ключи валидны).

## Текст для вставки в новую сессию

```bash
cd ~/projects/finbot-workspace  # та же директория где была первая сессия
claude --dangerously-skip-permissions
```

В чат:

```
Resume. Я тебя возобновляю после прерванной сессии.

Прочитай в таком порядке:

1. CLAUDE.md.
2. docs/STATE.md, там твоё последнее зафиксированное состояние.
3. docs/02_PLAYBOOK.md (только текущий и оставшиеся milestones).
4. docs/07_CHECKLISTS.md для текущего milestone.

Дальше:

1. Сверь реальное состояние с STATE.md:
   - git log --oneline -30
   - git status
   - supabase migration list
   - supabase functions list
   - psql -c "select jobname, schedule, active from cron.job" (если psql есть и БД настроена)
   - gh run list --workflow=deploy.yml --limit=3 (если CI настроен)

2. Если есть расхождения, приведи в консистентное состояние.

3. Продолжи с next_step из STATE.md.

4. Все правила из CLAUDE.md остаются в силе. Автономия по docs/00_AUTONOMY.md остаётся.

Все ключи лежат в .env, не запрашивай их у меня заново.

Старт resume.
```

## Что Claude Code сделает

1. Прочитает документы.
2. Запустит проверки состояния.
3. **Кратко** отчитается: "Resume с M7. git log в порядке, последний коммит feat(text): full text
   pipeline. cron jobs ещё не активны (это M14). Продолжаю с M8.".
4. Идёт дальше до v1.0.0.

## Если состояние сильно расходится

Например, Claude Code обнаружил что:

- В git есть коммит, которого нет в STATE.md (Серхий сам что-то коммитил).
- БД содержит миграции, которые не из миграционных файлов.
- Edge Functions задеплоены не те, что в коде.

Тогда Claude Code остановится с одним сообщением: "обнаружил расхождение, X в репо но Y в STATE. Что
делать?". Серхий говорит "trust git" / "trust STATE" / "вот контекст что было".

## Если STATE.md потерян

Маловероятно, но если:

```bash
git log --oneline | grep -E "(M\d+|feat|chore|fix)" | head -20
```

покажет последний милейстоун. Реконструируй STATE.md руками или через простой prompt:

```
STATE.md потерян. Восстанови его, используя git history. Последние 30 коммитов:
<paste git log output>

Текущий milestone определи по последнему conventional commit (feat(...), chore: ...) который соответствует одному из M1...M18 в docs/02_PLAYBOOK.md.

Запиши восстановленный STATE.md, потом продолжай работу.
```
