# Kickoff prompt, для первой сессии Claude Code

Этот файл содержит **точный текст**, который Серхий копирует в Claude Code (`claude` CLI в
директории `~/projects/finbot` или любой пустой папке) в самой первой сессии. Все документы пакета
уже должны лежать в этой директории, см. `README.md` пакета.

---

## Способ 1, через CLI с pre-authorized permissions

Запускаешь Claude Code так чтобы он не спрашивал разрешения на каждое действие:

```bash
cd ~/projects/finbot-workspace   # директория где лежат CLAUDE.md, SPEC.md, docs/
claude --dangerously-skip-permissions
```

Затем в чат вставляешь:

```
Привет. Прочитай файлы в этой директории в таком порядке:

1. CLAUDE.md, главный operational contract.
2. SPEC.md, источник правды по продукту.
3. docs/00_AUTONOMY.md, мой контракт автономии (даю тебе carte blanche на действия описанные там).
4. docs/01_BOOTSTRAP.md, что собирать с меня в первом сообщении.
5. docs/02_PLAYBOOK.md, M1...M18 step-by-step.
6. Остальные docs/*.md (03 conventions, 04 testing, 05 troubleshooting, 06 prompts, 07 checklists, 08 deploy, 09 recovery, 10 glossary).

Я тебя запустил в полностью автономном режиме. Действуй согласно CLAUDE.md и docs/00_AUTONOMY.md без дополнительных подтверждений. Не спрашивай разрешения на bash, git, gh, supabase CLI, файлы, сетевые запросы к разрешённым доменам.

Сейчас в одном сообщении запроси у меня все ключи и параметры (по списку из docs/01_BOOTSTRAP.md раздел 2). Я отвечу одним сообщением со всеми значениями, и дальше ты идёшь по M1...M18 до v1.0.0 без моего участия.

Останавливайся только по правилам из CLAUDE.md раздел "Когда останавливаться". Всё остальное решай сам через docs/05_TROUBLESHOOTING.md.

Старт.
```

## Способ 2, через `claude` без флага

Если по какой-то причине `--dangerously-skip-permissions` нельзя (например, на корпоративной
машине):

```bash
cd ~/projects/finbot-workspace
claude
```

В чат:

```
Привет. Прочитай CLAUDE.md и SPEC.md в этой директории. Также прочитай все docs/*.md.

Я даю тебе разрешение **сейчас, явно** делать без дополнительных вопросов:
- bash команды (git, gh, deno, supabase, curl, psql, npm, jq, age, openssl, make).
- создание, чтение, изменение файлов в этой директории и поддиректориях.
- сетевые запросы к: *.supabase.co, api.anthropic.com, api.groq.com, api.telegram.org, api.github.com, api.nbp.pl, api.exchangerate.host, registry.npmjs.org, deno.land, jsr.io, cdn.jsdelivr.net, cdnjs.cloudflare.com.

Если CC попросит approval на конкретное действие из перечисленных категорий - approve без меня (используй "all"/"yes-and-skip-similar" опции в TUI).

Сейчас в ОДНОМ сообщении запроси у меня все ключи (по списку из docs/01_BOOTSTRAP.md раздел 2). Я отвечу одним сообщением со всеми значениями, и дальше ты идёшь по M1...M18 до v1.0.0 без моего вмешательства.

Останавливайся только по правилам из CLAUDE.md раздел "Когда останавливаться".

Старт.
```

В этом режиме Claude Code будет иногда подтверждать через TUI prompt. Серхий нажимает "always" в
первый раз для каждого инструмента, дальше тихо.

## Что должно произойти после kickoff

1. Claude Code прочитает все документы (несколько Read tool calls).
2. Через ~30 секунд, отправит **одно сообщение** со списком из ~12 ключей и параметров (см.
   `docs/01_BOOTSTRAP.md` раздел 2).
3. Серхий копирует ответ от чат-партнёра, вставляет одним сообщением все ключи. Формат свободный.
4. Claude Code валидирует все ключи (curl проверки), создаёт `.env`, и начинает работу.
5. С этого момента Серхий не получает сообщений **за исключением** случаев из CLAUDE.md раздел
   "Когда останавливаться".
6. Финальное сообщение - отчёт о готовности v1.0.0 (см. CLAUDE.md раздел 7).

## Ожидаемое время работы

- Bootstrap (валидация ключей, setup tools): 5-10 минут.
- M1-M5 (skeleton, schema, idempotency, auth, claude+budget): 30-60 минут.
- M6-M11 (категоризатор, текст, голос, фото, media groups, edited): 60-120 минут.
- M12-M13 (api + webapp): 30-60 минут.
- M14-M15 (cron jobs + backup): 30-60 минут.
- M16-M18 (CI/CD + DR + docs): 30-60 минут.

**Итого: 3-7 часов** на одной длинной сессии Claude Code. Может быть несколько сессий с resume через
STATE.md.

## Что делать пользователю во время работы

Ничего. Можно закрыть терминал? **Нет**, Claude Code остановится. Терминал должен быть открыт.

Можно использовать tmux для надёжности:

```bash
tmux new -s finbot
cd ~/projects/finbot-workspace
claude --dangerously-skip-permissions
# ...вставить kickoff prompt...
# Ctrl-B D чтобы detach
```

Потом:

```bash
tmux attach -t finbot
```

Если соединение порвётся (laptop ушёл в сон) - tmux session жива, attach обратно. Если сама машина
перезагрузилась - Claude Code обрывается, но STATE.md есть, новая сессия подхватит.

## Если Claude Code запросил что-то посреди работы

См. CLAUDE.md раздел "Когда останавливаться". Это редкие случаи:

1. Ключ не работает - дай новый.
2. Внешний сервис недоступен > 10 мин - ждём.
3. SPEC противоречит сам себе - редкий случай, отвечай.
4. CI/CD сломан и три попытки не помогли - редкий случай, дай совет.
5. Контекст сессии закончился - открой новую: `claude` +
   `Прочитай CLAUDE.md и docs/STATE.md, продолжай где остановился`.
